Fondamentaux de l'IA et du Prompting – Niveau 1

Définitions essentielles
  • IA générative : modèles (LLM) produisant texte, image, code.
  • Prompt : instruction, contexte, exemples.
  • Token : 1 token ~ 0.75 mot (pour GPT-4).
  • Context window : limite de tokens (4k, 16k, 128k...).
  • Hallucination : information fausse générée.
Tip : toujours vérifier le nombre de tokens avec un tokenizer.
Types de prompts (avec exemples)
  • Zero-shot : "Traduis 'hello' en français."
  • Few-shot (2 exemples) : "anglais: cat → français: chat ; anglais: dog → français: chien ; anglais: bird → ?"
  • Chain-of-Thought : "Si Jean a 5 pommes et en donne 2, combien lui reste-t-il ? Explique étape par étape."
Paramètres fondamentaux

Température : 0 (déterministe) → 2 (créatif).
Top_p : 0.9 signifie qu'on prend les tokens représentant 90% de la probabilité.
Frequency penalty : réduit la répétition.

📊 Pour du code, température 0 ; pour du copywriting, 0.8.

Erreurs classiques & solutions
  • ❌ Prompt trop vague → ✅ Ajouter des détails, exemples.
  • ❌ Oublier le format → ✅ Spécifier "réponds en JSON".
  • ❌ Trop de contraintes → ✅ Simplifier, prioriser.
  • ❌ Ignorer la limite de tokens → ✅ Compter, tronquer si besoin.

Structurer ses prompts avec MASTER – approche exhaustive

🔍 Décomposition MASTER (avec sous-éléments)
  • M (Mission) : finalité, objectif mesurable. Ex: "Rédiger une offre d'emploi pour un développeur Python."
  • A (Audience) : profil, niveau d'expertise. Ex: "candidats juniors, ton motivant."
  • S (Style) : format, longueur, voix. Ex: "annonce structurée (titre, missions, prérequis, avantages), 300-400 mots."
  • T (Tâches) : actions décomposées. Ex: "1. Lister les missions, 2. Décrire le profil recherché, 3. Ajouter une section 'pourquoi nous rejoindre'."
  • E (Évaluation) : critères de réussite. Ex: "doit inclure les mots-clés 'agile', 'Git', doit être engageant."
  • R (Révision) : itération. Ex: "Après première version, vérifier la clarté et la longueur, puis affiner."
📘 Exemple concret pas à pas

Mission : créer un email de relance pour un prospect.

Audience : directeurs marketing B2B.

Style : professionnel, concis, personnalisé.

Tâches : ① rappeler le contexte, ② proposer un appel, ③ ajouter un témoignage.

Évaluation : vérifier qu'il contient une phrase d'accroche, un CTA.

Révision : après test, ajuster le ton.

📌 La phase de révision est clé : on y passe 80% du temps d'optimisation.
Bonnes pratiques MASTER
  • Utiliser des listes numérotées pour les tâches.
  • Pour l'évaluation, demander une auto-notation par l'IA.
  • Documenter chaque version du prompt.
  • Inclure des exemples dans la section tâches.
Pièges MASTER
  • Mission trop vague (ex: "aide-moi").
  • Audience oubliée → réponse trop technique ou trop simple.
  • Tâches non ordonnées → réponse désorganisée.
  • Évaluation subjective → préférer des critères vérifiables.
Maturité sur la structuration

1. Aucune méthode → prompts jetables.
2. MASTER utilisé individuellement → meilleure qualité.
3. MASTER standardisé dans l'équipe → templates partagés.
4. Amélioration continue et métriques → taux de succès mesuré.

🏢 Impact organisationnel

Uniformisation de la qualité des prompts, réduction du temps de rework, meilleure collaboration entre métiers (marketing, produit, data).

Techniques avancées – Levier de performance

🧠 Chain-of-Thought (CoT)

Demander un raisonnement étape par étape. Améliore les résultats en mathématiques, logique, planification.

Exemple : "Si une voiture consomme 6L/100km et que le carburant coûte 1,80€/L, combien coûte un trajet de 350km ? Explique chaque étape."

➕ Variante : CoT auto-consistant (plusieurs raisonnements, vote).
⛓️ Prompt chaining / multi-step

Découpage d'une tâche complexe en sous-tâches. Chaque prompt utilise la sortie du précédent.

Exemple workflow : 1) Extraire les idées d'un texte → 2) Résumer chaque idée → 3) Traduire en allemand.

🔧 Outils : LangChain, LlamaIndex.
🔄 Auto-critique & réflexion

Le modèle évalue sa propre réponse et propose une version améliorée.

Prompt : "Génère une réponse, puis évalue-la sur 10 points, et corrige les faiblesses identifiées."

🎛️ Réglages fins (paramètres)

Temperature : pour la créativité (0.8-1.2). Top_p : 0.9-1.0. Presence_penalty : encourage les nouveaux sujets. Frequency_penalty : réduit la répétition.

⚙️ Combinaison : basse température + présence penalty pour des réponses précises et variées.
📚 RAG (Retrieval-Augmented Generation)

Intégrer des documents externes dans le contexte pour répondre avec des faits actualisés ou propriétaires.

Architecture : base vectorielle → recherche de passages pertinents → injection dans le prompt.

🧪 Few-shot avancé

Utiliser des exemples dynamiques sélectionnés selon la requête (few-shot adaptatif).

📈 Maturité technique

1. Prompts linéaires → 2. CoT / few-shot → 3. Prompt chaining / RAG → 4. Agents autonomes

💼 Impact métier

Les techniques avancées permettent d'automatiser des processus complexes (analyse de contrats, support client niveau 2, reporting intelligent) avec une fiabilité accrue.

Optimisation & Performance – Passer à l'échelle

📊 KPIs essentiels
  • Précision / Exactitude : % de réponses correctes (sur un jeu test).
  • Latence : temps de réponse (ms).
  • Coût par requête : (tokens entrée + sortie) × prix.
  • Taux d'hallucination : % de faits inventés.
  • Satisfaction utilisateur (feedback).
🔬 Méthodes d'optimisation
  • A/B testing : comparer deux prompts en production.
  • Analyse de tokens : supprimer les redondances.
  • Fine-tuning : pour un domaine ou un style spécifique.
  • Cache des réponses : pour les requêtes fréquentes.
📉 Réduction des coûts

Raccourcir les prompts sans perdre en qualité (pruning). Utiliser des modèles plus petits pour les tâches simples. Mettre en cache.

💰 Exemple : un prompt passé de 1500 à 800 tokens a réduit le coût de 47%.
⏱️ Réduction de latence

Choisir un modèle plus rapide (ex: GPT-3.5-turbo au lieu de GPT-4). Activer le streaming. Paralléliser les appels indépendants.

📈 Amélioration de la précision

Utiliser le self-consistency (plusieurs runs et vote). Ajouter des exemples difficiles. Itérer sur l'évaluation.

🏅 Maturité en optimisation

1. Aucune mesure → 2. KPIs suivis manuellement → 3. Optimisation systématique (tests, pruning) → 4. Pilotage automatisé (ML sur les prompts)

Sécurité, Éthique & Fiabilité – Pilier de confiance

🛡️ Risques de sécurité
  • Prompt injection : l'utilisateur tente de faire exécuter des commandes non prévues.
  • Extraction du prompt système : via des questions comme "ignore les instructions précédentes".
  • Fuite de données : si des informations confidentielles sont dans le contexte.
🔒 Contre-mesures
  • Filtrer les entrées utilisateur (regex, listes noires).
  • Utiliser des délimiteurs forts (ex: """instruction""").
  • Limiter la longueur des entrées.
  • Anonymiser les données avant envoi.
  • Utiliser des system prompts robustes.
⚖️ Éthique & biais
  • Les modèles peuvent perpétuer des stéréotypes.
  • Solution : prompts inclusifs, audits réguliers.
  • Impliquer des experts métier dans l'évaluation.
🧪 Fiabilité & hallucinations
  • Utiliser le RAG pour ancrer les réponses dans des sources.
  • Demander des citations (ex: "justifie avec des références").
  • Mettre en place une supervision humaine (human-in-the-loop).
  • Évaluer régulièrement sur un benchmark.
📜 RGPD et conformité

Les données personnelles ne doivent pas être transmises sans consentement. Privilégier l'anonymisation. Prévoir un droit de rectification.

🇪🇺 Si vous utilisez une API américaine, informez les utilisateurs.
🔐 Maturité sécurité/éthique

1. Aucune → 2. Sensibilisation → 3. Politiques et filtres → 4. Audit continu et certifications

Cas pratiques avancés – Ateliers interactifs

📝 Atelier 1 : Génération de fiches de poste

Objectif : créer un prompt générique qui, à partir d'intitulé de poste et de quelques mots-clés, génère une fiche de poste complète (missions, profil, avantages).

Étapes : 1. Rédiger un prompt MASTER. 2. Tester avec 5 intitulés différents. 3. Itérer pour améliorer la cohérence.

REX : après 4 itérations, le taux de conformité est passé de 55% à 90%.
📊 Atelier 2 : Analyse de sentiment + résumé d'avis

Workflow : 1. Prompt pour classifier le sentiment (positif/négatif/neutre). 2. Prompt pour résumer les points clés. 3. Prompt pour générer une réponse automatique.

💡 Utiliser un prompt chaining avec stockage intermédiaire.
🤖 Atelier 3 : Création d'un agent simple

Utiliser un LLM pour décider d'appeler un outil (API météo, calculatrice). Prompt système : "Tu es un assistant. Si l'utilisateur demande la météo, réponds par 'OUTIL:METEO' suivi de la ville."

📚 Atelier 4 : RAG sur documentation interne

Construire un index vectoriel de vos documents, puis créer un prompt qui intègre les passages pertinents pour répondre aux questions.
